## 测试用例执行步骤

### 测试用例 1：AI 角色数据持久化

**目标**: 验证 AI 角色在重启后是否保留

**操作步骤**：

1. **创建 AI 角色**
   - 打开 AI Team 侧边栏
   - 点击"+ Add Member"创建 2 个角色：
     - 项目经理 PM，职能："项目管理"
     - 前端工程师 Alex，职能："Frontend Development"

2. **设置激活角色**
   - 点击项目经理卡片，确保其被选中（背景变蓝）

3. **重启 VS Code**
   - 完全退出 VS Code（Cmd+Q 或 File → Quit）
   - 重新启动 VS Code
   - 打开项目目录

4. **验证结果**
   - 打开 AI Team 侧边栏
   - 检查是否有 2 个角色
   - 检查角色信息是否完整（名称、职能）
   - 检查项目经理是否默认被选中（背景为蓝色）

**验收检查点**：
- [ ] 2 个角色都存在
- [ ] 角色名称正确
- [ ] 角色职能正确
- [ ] 项目经理默认被选中

## 调试技巧

### 查看存储的数据

在浏览器开发者工具 Console 中执行：

```javascript
// 查看存储的会话数据
localStorage.getItem('vscode.workspace.humancode.sessions')

// 查看存储的消息数据
localStorage.getItem('vscode.workspace.humancode.messages')

// 查看存储的上下文数据
localStorage.getItem('vscode.workspace.humancode.context')
```

**注意**: VS Code 的 `IStorageService` 可能使用不同的存储机制，上述代码仅供参考。

### 清除所有数据

如果需要重新开始测试：

1. 关闭 VS Code
2. 删除存储文件（具体位置取决于 VS Code 的存储实现）
3. 重新启动 VS Code

## 常见问题排查

### 问题 1: 重启后数据丢失

**可能原因**:
- 存储服务未正确初始化
- 存储路径错误
- 权限问题

**排查步骤**:
1. 检查浏览器开发者工具 Console 是否有错误
2. 查找包含 "Failed to save" 或 "Failed to load" 的日志
3. 检查存储服务是否正确注入

### 问题 2: 部分数据丢失

**可能原因**:
- 增量保存失败
- 数据序列化错误

**排查步骤**:
1. 检查 Console 日志
2. 验证数据格式是否正确
3. 检查是否有 JSON 序列化错误

### 问题 3: 启动变慢

**可能原因**:
- 数据量过大
- 加载逻辑效率低

**排查步骤**:
1. 检查存储的数据大小
2. 考虑清理旧数据
3. 等待性能优化实现
