**文档版本**: v1.0
**创建日期**: 2026-03-25
**状态**: 设计方案

---

## 一、重构背景

### 1.1 当前界面的问题

经过 Phase 1 和 Phase 2 的开发，当前界面存在以下问题：

1. **输入框功能不完整**
   - 没有明确显示当前对话的目标角色
   - 缺少发送状态反馈
   - 没有历史消息显示

2. **配置界面缺失**
   - 创建角色时没有选择 AI 扩展的 UI
   - 没有编辑角色配置的界面
   - 缺少全局设置面板

3. **消息聊天界面简陋**
   - 消息汇总面板只是简单的列表
   - 缺少消息分组和筛选
   - 没有对话上下文展示

4. **角色管理界面不完善**
   - 角色卡片信息展示不够清晰
   - 缺少角色详情查看
   - 没有角色编辑和删除的便捷入口

### 1.2 设计目标

根据 HumanCode 的产品定位："让没有编程知识的普通人也能通过 AI 团队协作平台独立完成软件开发"，界面重构应遵循以下原则：

1. **零门槛**：所有操作用自然语言描述，不暴露技术术语
2. **状态透明**：AI 在做什么、做完了什么，用户随时一眼看清
3. **渐进式复杂度**：基础功能极简，高级功能按需展开
4. **拟人化**：AI 角色用"成员卡片"而非"进程列表"来呈现

### 3.2 角色卡片设计

**卡片元素**：
1. **选中指示器**：蓝色圆点（选中）/ 灰色圆点（未选中）
2. **角色名称**：大号字体，易于识别
3. **状态指示**：
   - 空闲：灰色文字
   - 工作中：绿色文字 + 绿色圆点 + 呼吸动画
   - 等待：黄色文字 + 黄色圆点
   - 出错：红色文字 + 红色圆点
4. **绑定扩展**：显示当前角色使用的 AI 扩展
5. **操作按钮**：
   - [详情]：查看角色详细信息
   - [编辑]：编辑角色配置

**交互行为**：
- 点击卡片 → 选中该角色（卡片变蓝）
- 双击卡片 → 打开角色详情
- 右键菜单 → 编辑、删除、复制配置
