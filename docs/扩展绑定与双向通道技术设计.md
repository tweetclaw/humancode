## 一、设计目标

实现虚拟 AI 员工与真实 AI 扩展之间的双向通信通道，使用户能够：

1. **列出已安装的 AI 扩展**：自动检测系统中已安装的 AI 扩展（通义灵码、Copilot 等）
2. **绑定扩展到角色**：创建虚拟员工时，选择并绑定到具体的 AI 扩展
3. **主动发送消息**：从 HumanCode 输入框发送消息给绑定的 AI 扩展
4. **接收响应**：拦截 AI 扩展的响应并记录到会话历史

## 三、技术方案设计

### 3.1 获取已安装的 AI 扩展列表

#### 方案概述

利用 VS Code 的 `IExtensionService` 获取所有已安装的扩展，并过滤出 AI 相关的扩展。

#### 实现位置

新增服务：`src/vs/workbench/services/aiExtensions/common/aiExtensionRegistry.ts`

#### 接口设计

```typescript
/**
 * AI 扩展信息
 */
export interface IAIExtensionInfo {
	/** 扩展 ID，如 "alibaba-cloud.tongyi-lingma" */
	readonly id: string;
	/** 显示名称，如 "通义灵码" */
	readonly displayName: string;
	/** 扩展描述 */
	readonly description: string;
	/** 扩展版本 */
	readonly version: string;
	/** 是否已激活 */
	readonly isActive: boolean;
	/** 扩展图标 URI（可选） */
	readonly iconUri?: URI;
}

/**
 * AI 扩展注册表服务
 */
export interface IAIExtensionRegistryService {
	readonly _serviceBrand: undefined;

	/**
	 * 获取所有已安装的 AI 扩展
	 */
	getAllAIExtensions(): Promise<IAIExtensionInfo[]>;

	/**
	 * 获取已激活的 AI 扩展
	 */
	getActiveAIExtensions(): Promise<IAIExtensionInfo[]>;

	/**
	 * 检查扩展是否为 AI 扩展
	 */
	isAIExtension(extensionId: string): boolean;

	/**
	 * 事件：AI 扩展列表变化
	 */
	readonly onDidChangeAIExtensions: Event<void>;
}
```

#### 实现逻辑

```typescript
export class AIExtensionRegistryService implements IAIExtensionRegistryService {

	// 已知的 AI 扩展 ID 列表
	private readonly KNOWN_AI_EXTENSIONS = [
		'alibaba-cloud.tongyi-lingma',
		'github.copilot',
		'github.copilot-chat',
		'continue.continue',
		'tabnine.tabnine-vscode',
		// ... 可扩展
	];

	constructor(
		@IExtensionService private readonly extensionService: IExtensionService
	) {}

	async getAllAIExtensions(): Promise<IAIExtensionInfo[]> {
		const allExtensions = await this.extensionService.getExtensions();

		return allExtensions
			.filter(ext => this.isAIExtension(ext.identifier.value))
			.map(ext => ({
				id: ext.identifier.value,
				displayName: ext.displayName || ext.name,
				description: ext.description || '',
				version: ext.version,
				isActive: this.extensionService.activateByEvent(`onExtension:${ext.identifier.value}`),
				iconUri: ext.icon
			}));
	}

	isAIExtension(extensionId: string): boolean {
		// 方法 1：检查已知列表
		if (this.KNOWN_AI_EXTENSIONS.includes(extensionId)) {
			return true;
		}

		// 方法 2：检查扩展的 package.json 中的关键词
		// 如果扩展的 keywords 包含 'ai', 'copilot', 'assistant' 等
		const extension = this.extensionService.getExtension(extensionId);
		if (extension) {
			const keywords = extension.keywords || [];
			const aiKeywords = ['ai', 'copilot', 'assistant', 'chat', 'code-generation'];
			return keywords.some(k => aiKeywords.includes(k.toLowerCase()));
		}

		return false;
	}
}
```

### 3.2 扩展绑定机制

#### 方案概述

在创建虚拟员工时，让用户选择要绑定的 AI 扩展，并将 `extensionId` 保存到会话配置中。

#### UI 流程

```
用户点击"添加成员"
    ↓
步骤 1：选择角色模板（项目经理、前端工程师等）
    ↓
步骤 2：输入角色名称
    ↓
步骤 3：选择 AI 扩展  ← 新增步骤
    ↓
步骤 4：编辑系统提示词（可选）
    ↓
创建完成
```

#### 实现位置

修改：`src/vs/workbench/contrib/aiTeam/browser/aiTeamPanel.ts`

#### 实现代码

```typescript
private async createNewSession(): Promise<void> {
	// 步骤 1：选择角色模板
	const roleTemplate = await this.quickInputService.pick([
		{ label: '项目经理', description: '负责需求分析和任务分配' },
		{ label: '前端工程师', description: '负责前端开发' },
		{ label: '后端工程师', description: '负责后端开发' },
		{ label: '测试工程师', description: '负责测试和质量保证' },
		{ label: '自定义角色', description: '创建自定义角色' }
	], { placeHolder: '选择角色类型' });

	if (!roleTemplate) { return; }

	// 步骤 2：输入角色名称
	const name = await this.quickInputService.input({
		prompt: '输入角色名称',
		placeHolder: '例如：前端小李'
	});

	if (!name) { return; }

	// 步骤 3：选择 AI 扩展（新增）
	const aiExtensions = await this.aiExtensionRegistry.getAllAIExtensions();

	if (aiExtensions.length === 0) {
		this.notificationService.error('未检测到已安装的 AI 扩展，请先安装通义灵码或 GitHub Copilot');
		return;
	}

	const selectedExtension = await this.quickInputService.pick(
		aiExtensions.map(ext => ({
			label: ext.displayName,
			description: ext.description,
			detail: `版本: ${ext.version} | ID: ${ext.id}`,
			extensionId: ext.id
		})),
		{ placeHolder: '选择要使用的 AI 扩展' }
	);

	if (!selectedExtension) { return; }

	// 步骤 4：生成系统提示词
	const systemPrompt = this.generateSystemPrompt(roleTemplate.label);

	// 创建会话
	const sessionId = this.sessionManager.createSession({
		name,
		role: roleTemplate.label,
		extensionId: selectedExtension.extensionId,  // ← 绑定扩展
		systemPrompt
	});

	this.notificationService.info(`已创建角色 "${name}"，绑定到 ${selectedExtension.label}`);
}
```

### 3.3 主动发送消息给 AI 扩展

#### 方案概述

研究 AI 扩展的 RPC 消息格式，实现通用的消息发送机制。

#### 技术挑战

不同 AI 扩展的 RPC 消息格式不同：

**通义灵码示例**（需要实际抓包验证）:
```json
{
  "method": "tongyi/chat",
  "params": {
    "message": "实现登录功能",
    "context": "...",
    "sessionId": "..."
  }
}
```

**GitHub Copilot 示例**（需要实际抓包验证）:
```json
{
  "method": "copilot/sendMessage",
  "params": {
    "prompt": "实现登录功能",
    "conversationId": "..."
  }
}
```

#### 实现策略

**方案 A：扩展适配器模式**（推荐）

为每个 AI 扩展实现一个适配器，封装消息格式的差异。

```typescript
/**
 * AI 扩展适配器接口
 */
export interface IAIExtensionAdapter {
	/**
	 * 扩展 ID
	 */
	readonly extensionId: string;

	/**
	 * 发送消息给 AI 扩展
	 */
	sendMessage(message: string, context: string): Promise<void>;

	/**
	 * 检测是否为该扩展的响应消息
	 */
	isResponseMessage(data: any): boolean;

	/**
	 * 提取响应内容
	 */
	extractResponse(data: any): string;
}

/**
 * 通义灵码适配器
 */
export class TongyiLingmaAdapter implements IAIExtensionAdapter {
	readonly extensionId = 'alibaba-cloud.tongyi-lingma';

	constructor(
		@IRPCProtocol private readonly rpcProtocol: IRPCProtocol
	) {}

	async sendMessage(message: string, context: string): Promise<void> {
		// 构造通义灵码的 RPC 消息格式
		await this.rpcProtocol.send({
			method: 'tongyi/chat',  // ⚠️ 需要实际验证
			params: {
				message,
				context,
				sessionId: generateId()
			}
		});
	}

	isResponseMessage(data: any): boolean {
		return data?.method === 'tongyi/chatResponse';  // ⚠️ 需要实际验证
	}

	extractResponse(data: any): string {
		return data?.params?.response || '';  // ⚠️ 需要实际验证
	}
}
```

#### 集成到会话管理服务

修改：`src/vs/workbench/services/aiSessionManager/browser/aiSessionManagerService.ts`

```typescript
export class AISessionManagerService implements IAISessionManagerService {

	constructor(
		@IAIExtensionAdapterRegistry private readonly adapterRegistry: IAIExtensionAdapterRegistry,
		// ... 其他依赖
	) {}

	/**
	 * 发送消息给 AI 扩展
	 */
	async sendMessageToAI(sessionId: string, userMessage: string): Promise<void> {
		const session = this._sessions.get(sessionId);
		if (!session) {
			throw new Error(`Session ${sessionId} not found`);
		}

		// 获取对应的扩展适配器
		const adapter = this.adapterRegistry.getAdapter(session.extensionId);
		if (!adapter) {
			throw new Error(`No adapter found for extension ${session.extensionId}`);
		}

		// 构造上下文
		const context = this.getSessionContext(sessionId);

		// 更新状态为 working
		this.updateSessionStatus(sessionId, 'working');

		// 记录用户消息
		this.appendMessage(sessionId, {
			id: generateId(),
			direction: 'user',
			content: userMessage,
			timestamp: Date.now()
		});

		try {
			// 发送消息给 AI 扩展
			await adapter.sendMessage(userMessage, context);
		} catch (error) {
			this.updateSessionStatus(sessionId, 'error');
			throw error;
		}
	}
}
```

### 3.4 接收并记录 AI 响应

#### 方案概述

在 `HumanCodeRPCLogger.logIncoming()` 中，使用适配器检测并提取 AI 响应。

#### 实现位置

修改：`src/vs/workbench/services/extensions/common/extensionHostManager.ts`

#### 实现代码

```typescript
class HumanCodeRPCLogger implements IRPCProtocolLogger {

	constructor(
		@IAISessionManagerService private readonly sessionManager: IAISessionManagerService,
		@IAIExtensionAdapterRegistry private readonly adapterRegistry: IAIExtensionAdapterRegistry
	) {}

	logIncoming(str: string, data?: any): void {
		// 原有的日志记录
		this.extensionMessagesService.logMessage({
			direction: 'incoming',
			str,
			data,
			timestamp: Date.now()
		});

		// 检测是否为 AI 响应
		if (this.sessionManager && data) {
			const activeSessionId = this.sessionManager.getActiveSessionId();
			if (activeSessionId) {
				const session = this.sessionManager.getSession(activeSessionId);
				if (session) {
					// 获取对应的适配器
					const adapter = this.adapterRegistry.getAdapter(session.extensionId);
					if (adapter && adapter.isResponseMessage(data)) {
						// 提取响应内容
						const response = adapter.extractResponse(data);

						// 记录到会话历史
						this.sessionManager.appendMessage(activeSessionId, {
							id: generateId(),
							direction: 'assistant',
							content: response,
							timestamp: Date.now()
						});

						// 更新状态为 idle
						this.sessionManager.updateSessionStatus(activeSessionId, 'idle');
					}
				}
			}
		}
	}
}
```

## 四、完整的数据流向

### 4.1 用户在 HumanCode 输入框发送消息

```
用户在 HumanCode 输入框输入："实现登录功能"
    ↓
AI Team Panel 调用 sessionManager.sendMessageToAI(sessionId, message)
    ↓
AISessionManagerService:
  1. 获取会话的 extensionId
  2. 获取对应的适配器
  3. 构造上下文（系统提示词 + 历史记录）
  4. 更新状态为 'working'
  5. 记录用户消息
    ↓
适配器（如 TongyiLingmaAdapter）:
  1. 构造 RPC 消息（注入上下文）
  2. 通过 RPCProtocol 发送给通义灵码
    ↓
通义灵码处理消息并返回响应
    ↓
HumanCodeRPCLogger.logIncoming():
  1. 检测是否为 AI 响应
  2. 使用适配器提取响应内容
  3. 记录到会话历史
  4. 更新状态为 'idle'
    ↓
UI 更新：显示 AI 响应
```

## 六、技术风险与缓解

### 6.1 风险：AI 扩展的 RPC 消息格式未知

**影响**: 无法构造正确的 RPC 消息

**缓解措施**:
1. 使用 Extension Messages View 抓包分析
2. 阅读 AI 扩展的源代码（如果开源）
3. 逐步试错，验证消息格式

---

### 6.2 风险：不同版本的 AI 扩展消息格式不同

**影响**: 适配器可能失效

**缓解措施**:
1. 适配器支持版本检测
2. 为不同版本实现不同的适配逻辑
3. 提供降级方案（如果无法识别，提示用户）

---

### 6.3 风险：AI 扩展不支持外部调用

**影响**: 无法主动发送消息

**缓解措施**:
1. 如果 AI 扩展不支持外部调用，回退到"拦截模式"
2. 用户仍然在 AI 扩展的原生窗口输入
3. HumanCode 只负责拦截和注入上下文
