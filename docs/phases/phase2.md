## 一、阶段目标

**核心定位**：提供"消息聚合与转发平台"，让用户通过消息转发控制 AI 团队的协作流程。

### 我们做什么

- ✅ **团队列表与选中机制**：显示所有 AI 角色，点击选中，输入框内容发送给选中角色
- ✅ **消息汇总面板**：显示所有交互消息（时间线视图），每条消息有转发按钮
- ✅ **消息转发机制**：将消息转发给其他 AI 角色，目标角色自动被选中
- ✅ **全局上下文库**：跨会话共享项目记忆，简化 AI 重新整理思路的过程

### 我们不做什么

- ❌ **需求分析**：这是 AI 项目经理的工作，不是 IDE 的工作
- ❌ **任务拆分**：AI 项目经理根据需求自主拆分任务
- ❌ **角色分配**：AI 项目经理决定将任务分配给哪个 AI 角色
- ❌ **智能决策**：所有决策由 AI 自己完成，IDE 不做关键词匹配或规则引擎

### 本阶段结束后，用户应当能够

1. 在团队列表中看到所有 AI 角色，默认选中项目经理
2. 向选中的 AI 角色发送消息
3. 在消息汇总面板中看到所有交互消息
4. 点击消息的"转发"按钮，将消息转发给其他 AI 角色
5. 转发后，目标角色自动被选中，可以继续对话
6. 所有 AI 角色共享项目的全局上下文

## 三、详细设计

### 3.1 团队列表与选中机制

**UI 结构**：
```
┌─────────────────────┐
│  AI 团队            │
├─────────────────────┤
│  [项目经理 PM] 🔵   │  ← 默认选中（蓝色）
│  状态：空闲          │
│                     │
│  [前端工程师 Alex]  │
│  状态：工作中        │
│                     │
│  [测试工程师 Jordan]│
│  状态：空闲          │
└─────────────────────┘
```

**功能**：
- 显示所有 AI 角色（从 `IAISessionManagerService` 获取）
- 默认选中项目经理（卡片变蓝）
- 点击角色卡片 → 选中该角色（卡片变蓝）
- 当前选中的角色 = 聊天输入框的目标对象
- 显示角色状态（空闲、工作中、等待）

**输入窗口设计**：
```
┌─────────────────────────────────────────────────┐
│  正在与：项目经理 PM 对话                        │
├─────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────┐  │
│  │ 请输入消息...                              │  │
│  │                                            │  │
│  │                                            │  │
│  └───────────────────────────────────────────┘  │
│                                    [发送 ↑]     │
└─────────────────────────────────────────────────┘
```

**输入窗口功能**：
- 显示当前选中的目标角色（"正在与：XXX 对话"）
- 多行文本输入框
- 发送按钮（或按 Enter 发送）
- 输入的内容发送给当前选中的角色
- 切换选中角色后，输入框的目标自动更新

**实现要点**：
- 使用 Phase 1 已完成的 `AI Team Panel`
- 增加选中状态管理
- 选中状态变化时，更新 `activeSessionId`
- 输入窗口位于 AI Team Panel 底部
- 输入框上方显示当前目标角色名称

### 3.2 消息汇总面板

**UI 结构**：
```
┌──────────────────────────────────────────────────┐
│  消息汇总                                [过滤]   │
├──────────────────────────────────────────────────┤
│  [用户 → 项目经理 PM] 10:30                      │
│  实现用户登录功能                                 │
│  └─ [转发]                                       │
│                                                   │
│  [项目经理 PM → 用户] 10:31                      │
│  收到！我会安排前端工程师实现登录页面...          │
│  └─ [转发]                                       │
│                                                   │
│  [用户 → 前端工程师 Alex] 10:35                  │
│  请实现登录页面，使用 React                       │
│  └─ [转发]                                       │
│                                                   │
│  [前端工程师 Alex → 用户] 10:40                  │
│  已完成登录页面，使用了 React + TypeScript...    │
│  └─ [转发]                                       │
└──────────────────────────────────────────────────┘
```

**展示内容**：
1. 用户与 AI 项目经理的交互信息
2. 发送给各个 AI 的指令
3. AI 通过大模型处理后的**最终响应内容**（不包含推理过程）

**关键特性**：
- 每个角色只知道用户给他的信息（会话隔离）
- 每条消息显示：发送者 → 接收者、时间、内容
- 每条消息有"转发"按钮
- 支持过滤（按角色、时间、关键词）
- 只显示最终响应，不显示推理过程

**实现要点**：
- 监听 `IAISessionManagerService.onDidMessageAppend` 事件
- 将所有消息汇总到统一面板
- 消息格式：`{ from: string, to: string, content: string, timestamp: number }`

### 3.3 消息转发机制

**转发 UI**：
```
点击"转发"按钮后弹出：

┌─────────────────────────────┐
│  转发消息                    │
├─────────────────────────────┤
│  选择目标角色：              │
│  ○ 项目经理 PM              │
│  ○ 前端工程师 Alex          │
│  ● 测试工程师 Jordan        │
│                             │
│  [取消]  [确认转发]         │
└─────────────────────────────┘
```

**转发流程**：
```
用户在消息面板看到某条消息
    ↓
点击该消息的"转发"按钮
    ↓
选择目标角色（从已创建的角色列表中选择）
    ↓
消息转发给目标角色
    ↓
目标角色自动被选中（团队列表中变蓝）
    ↓
聊天输入框的目标切换到该角色
```

**使用场景**：
- **场景 1**：AI 项目经理的任务描述 → 转发给前端工程师
- **场景 2**：前端工程师写完代码有问题 → 直接选中他对话 → 满意后转发给项目经理
- **场景 3**：项目经理收到反馈 → 分解下一个任务

**实现要点**：
- 调用 `IAISessionManagerService.relayMessage()` 转发消息
- 转发后，调用 `setActiveSession()` 切换选中角色
- 更新团队列表的选中状态

### 3.4 全局上下文库

**目的**：简化每个 AI 重新整理代码和思路的过程

**功能**：
- 跨会话共享项目记忆（技术栈、架构决策、已实现功能）
- Token 预算管理
- 上下文检索

**实现要点**：
- 使用 `IStorageService` 持久化上下文
- 提供 `addContext()`, `getContext()`, `retrieveContext()` 方法
- 在 `getSessionContext()` 中注入全局上下文

**详细设计**：参考原 Phase 2 的 ContextLibrary 设计（保留）

## 七、关键决策记录

**2026-03-25 — Phase 2 重新设计**

**问题**：原设计让 IDE 替 AI 做决策（TaskDispatcher 使用关键词匹配分析需求、拆分任务、分配角色），违背产品定位。

**决策**：
1. **删除 TaskDispatcher 的智能分析功能**：这是 AI 项目经理的工作，不是 IDE 的工作
2. **用户控制任务调度**：第一版让用户通过消息转发控制任务流转，确保可靠性
3. **消息转发 + 自动选中**：简化操作流程，转发后无需再手动点击团队列表
4. **会话隔离**：每个角色只知道用户给他的信息，保持上下文清晰
5. **只显示最终响应**：不显示大模型推理过程，只关心结果
6. **保留 ContextLibrary**：这是合理的基础设施，不涉及智能决策
7. **删除 WorkflowEditor**：Phase 2 不需要可视化工作流编辑器，这是 Phase 3 的内容

**核心理念**：
- IDE 提供的是"消息聚合与转发平台"，不是"智能决策系统"
- AI 的能力远超关键词匹配，让 AI 自己做决策更智能
- IDE 只需提供基础设施，不越界做 AI 的工作
