# 开发环境配置

## 启动脚本

**主启动脚本**: `/Users/immeta/work/humancode/startcode.sh`

```bash
./startcode.sh
```

该脚本会:
- 清空日志文件 `1.log`
- 使用 `--log trace` 级别启动 VS Code
- 使用独立的用户数据目录和扩展目录(避免污染正常的 VS Code 环境)
- 将所有日志输出到 `1.log` 文件
- 过滤掉一些噪音日志(ResizeObserver、geminicodeassist 等)

## 日志文件

**运行时日志**: `/Users/immeta/work/humancode/1.log`

所有运行时的日志都会输出到这个文件,包括:
- RPC 消息拦截日志
- 扩展通信日志
- 调试信息
- 错误信息

查看日志:
```bash
# 实时查看日志
tail -f 1.log

# 搜索特定关键词
grep "HumanCodeRPCLogger" 1.log
grep "logOutgoing" 1.log
```

## 自动编译(Watch 模式)

### 方式一:使用 VS Code 内置任务(推荐)

在 VS Code 中按 `Cmd+Shift+B` 选择 `VS Code - Build` 任务,该任务会:
- 自动监控 `src/` 目录下的 TypeScript 文件变化
- 自动监控 `extensions/` 目录下的内置扩展变化
- 增量编译,速度快
- 在 VS Code 的 Terminal 面板中显示编译结果

### 方式二:使用命令行

```bash
# 监控主代码(src/)的变化并自动编译
npm run watch

# 或者使用 gulp 任务
npm run gulp watch
```

### 编译检查

```bash
# 仅检查 TypeScript 类型错误(不生成文件,速度快)
npm run compile-check-ts-native

# 编译扩展
npm run gulp compile-extensions

# 检查分层架构
npm run valid-layers-check
```

## 开发工作流

### 标准开发流程

1. **启动 Watch 模式**
   ```bash
   # 在 VS Code 中按 Cmd+Shift+B,选择 "VS Code - Build"
   ```

2. **启动开发环境**
   ```bash
   ./startcode.sh
   ```

3. **修改代码**
   - Watch 模式会自动编译
   - 在开发窗口中按 `Cmd+R` 重新加载窗口以应用更改

4. **查看日志**
   ```bash
   tail -f 1.log
   ```

### 调试技巧

1. **查看 RPC 消息**
   - 在开发窗口中按 `Cmd+Option+I` 打开 DevTools
   - 切换到 Console 标签页
   - 查看 `[HumanCodeRPCLogger]` 开头的日志

2. **查看编译错误**
   - 如果使用 VS Code 任务,错误会显示在 Terminal 面板
   - 如果使用命令行,错误会直接输出到终端

3. **清理构建产物**
   ```bash
   # 清理编译输出
   npm run gulp clean

   # 重新编译
   npm run compile
   ```

## 常见问题

### Q: 修改代码后没有生效?
A:
1. 确认 Watch 模式正在运行
2. 检查是否有编译错误
3. 在开发窗口中按 `Cmd+R` 重新加载

### Q: 找不到日志输出?
A:
1. 确认使用 `./startcode.sh` 启动(不是 `./scripts/code.sh`)
2. 检查 `1.log` 文件是否存在
3. 使用 `tail -f 1.log` 实时查看

### Q: 编译很慢?
A:
1. 使用 Watch 模式而不是每次手动编译
2. 使用 `npm run compile-check-ts-native` 仅检查类型(不生成文件)
3. 只修改必要的文件,避免触发大范围重新编译
